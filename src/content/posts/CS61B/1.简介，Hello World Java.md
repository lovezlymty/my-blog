## 1. 第一个 Java 程序
public class HelloWorld {
    public static void main(String[] args) {
        System.out.println("Hello world!");
    }
}
从这个程序可以看出 Java 的基本规则：
- 所有代码都位于类中；
- 程序从 `main` 方法开始执行；
- `{}` 表示类、方法等代码块的范围；
- 普通语句以分号结束；
- 变量使用前必须声明，并写明类型；
- 方法必须声明返回类型，没有返回值时使用 `void`。
`main` 的完整声明暂时可以当作固定入口理解：
public static void main(String[] args)
其中 `String[] args` 用来接收命令行参数；`public`、`static` 等关键字会在后续课程中逐步解释。

## 2. Java 的静态类型
Java 要求变量、参数和方法返回值都具有明确的类型。例如：
int x = 5;
x = "hello"; // 编译错误：String 不能赋给 int
这里的错误会在程序运行前被编译器发现。静态类型的主要好处是：
- 很多类型错误能在运行前暴露；
- 变量和方法的用途更容易从声明中看出来；
- 大型程序更容易理解、重构和维护；
- 编译成功的程序通常不会再出现类型不匹配问题。
代价是代码比动态类型语言更啰嗦，有时也会限制写法。但在需要长期维护的大型工程中，明确的类型通常能降低理解和修改代码的成本。

## 3. 编译与运行
Java 源代码不能直接由 `java` 命令运行，传统流程分为两步：
javac HelloWorld.java
java HelloWorld
第一条命令调用编译器，将 `HelloWorld.java` 编译为 `HelloWorld.class`；第二条命令启动 JVM，执行编译后的类。修改源代码后，需要重新编译，否则运行的仍可能是旧的 `.class` 文件。
可以把这一过程记成：
HelloWorld.java → javac → HelloWorld.class → java/JVM → 运行结果
